Parameter Anatomy

一层 Transformer 的参数是怎么构成的

每一层都由「注意力」和「FFN」两块组成。下面用一个具体例子把每个权重矩阵拆开,你能看清每份参数从哪来、怎么算。例子取 d=4096，H=32，head_dim=128，kv=8。

总装公式

Embedding + L × ( 注意力 + FFN ) Embedding 在层外,只加一次(tie 决定 ×1 还是 ×2);括号里的一层要 × 层数 L。norm / router 很小,决定量级时可忽略。

① 注意力块 · 每层 4 个投影

Attention 每个投影 = 输入维 × 输出维
q_proj d=4096→H·dₕ = 32×128 = 4096 16.8M
k_proj d=4096→kv·dₕ = 8×128 = 1024 4.2M
v_proj d=4096→kv·dₕ = 8×128 = 1024 4.2M
o_proj H·dₕ = 4096→d=4096 16.8M
GQA 的关键:q / o 用全部 32 个头,k / v 只有 8 个 kv 头,所以 k / v 各只有 q 的 1/4。永远别漏 head_dim——输出维是「头数 × head_dim」,不是头数本身。
一层注意力小计≈ 42.0M

② FFN 块 · 两种形态(二选一)

Dense FFN gated → 3 个矩阵
gated → Id×I
upd → Id×I
downI → dI×d
两个升维(gate、up)+ 一个降维(down)。silu/gelu 门控一律 ×3;老式 ReLU MLP 只有 up+down = ×2。
合计3 × d × I
MoE FFN 多个专家 + 挑选器
路由专家 ×N
3 × d × moe_I × N 大头
共享专家 ×S
3 × d × shared_I × S 每次都走
router / gate
d × N 很小
每个专家自己就是一个 gated FFN(×3、×d 都不能漏)。专家 FFN 用 moe_intermediate_size,通常比 dense 的 I 小很多,靠数量取胜。
路由专家(专精,选 top-k 才走) 共享专家(通用,每 token 必走) router(挑专家的打分器,dense 没有)

③ MoE 的两个数:总参数 vs 激活参数

TOTAL 总参数
全部 N 个专家

不做筛选,把所有专家都算进去。决定模型「有多大 / 占多少显存」。

路由 ×N + 共享 ×S + router
ACTIVATED 激活参数
只有 top-k 个专家

每个 token 实际走过的。决定「算得多快 / 单次前向的计算量」。

路由 ×k + 共享 ×S + router + 注意力

两者唯一的区别:总参数用 N(全部专家),激活参数用 k(top-k)。注意力、共享专家、router、dense 层在两个数里都全额计入。这就是 31B-A3B 这种写法的来历。

三条肌肉记忆
1. 每个矩阵 = 输入维 × 输出维,注意力别漏 head_dim,专家别漏 d。
2. gated FFN = 3 个矩阵(gate/up/down);router 和共享专家是 MoE 专有,dense 没有。
3. 算完一层立刻 × 层数,再把层外的 embedding 加上(tie=false 则 ×2)。
